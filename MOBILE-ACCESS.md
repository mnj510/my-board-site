# 📱 모바일 접속 가이드

## ⚠️ 중요: localhost는 모바일에서 작동하지 않습니다!

### 문제점
- `localhost` 또는 `127.0.0.1`은 **같은 기기에서만** 접속 가능합니다
- 모바일에서 `localhost`를 입력하면 모바일 기기 자체를 가리킵니다
- 서버가 실행되는 컴퓨터(Mac mini)의 IP 주소를 사용해야 합니다

## ✅ 올바른 접속 방법

### 서버 IP 주소 확인

터미널에서 다음 명령어로 확인:
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

또는 간단하게:
```bash
hostname -I
```

### 모바일에서 접속

**❌ 잘못된 방법:**
```
http://localhost:3000
```

**✅ 올바른 방법:**
```
http://[서버IP주소]:3000
```

예시:
```
http://192.168.219.46:3000
```

## 📋 접속 체크리스트

### 1. 서버가 실행 중인지 확인
```bash
lsof -ti:3000
```
프로세스 ID가 나오면 서버가 실행 중입니다.

### 2. 같은 Wi-Fi 네트워크에 연결
- ✅ Mac mini와 모바일이 같은 Wi-Fi에 연결되어 있어야 합니다
- ❌ 다른 네트워크에서는 접속 불가

### 3. 서버 IP 주소 확인
- Mac mini의 IP 주소를 확인하세요
- 모바일에서 해당 IP로 접속하세요

### 4. 방화벽 설정 확인
- macOS 방화벽에서 포트 3000 허용
- 시스템 설정 > 네트워크 > 방화벽

## 🔧 문제 해결

### 모바일에서 접속이 안 될 때

1. **서버 IP 확인**
   ```bash
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```

2. **서버 실행 확인**
   ```bash
   lsof -ti:3000
   ```

3. **같은 Wi-Fi 확인**
   - Mac mini와 모바일이 같은 Wi-Fi에 연결되어 있는지 확인

4. **방화벽 확인**
   - 시스템 설정 > 네트워크 > 방화벽
   - 포트 3000 허용

5. **서버 재시작**
   ```bash
   cd /Users/nj/Downloads/site2
   npm run dev
   ```

### 브라우저 오류 메시지

- **"연결할 수 없음"**: 서버가 실행 중이 아니거나 IP 주소가 잘못됨
- **"타임아웃"**: 방화벽이 차단하거나 다른 네트워크에 연결됨
- **"연결 거부됨"**: 서버가 네트워크 접속을 허용하지 않음

## 🚀 빠른 해결

### 1단계: 서버 IP 확인
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

### 2단계: 모바일에서 접속
브라우저 주소창에 입력:
```
http://[위에서 확인한 IP]:3000
```

예시:
```
http://192.168.219.46:3000
```

### 3단계: 접속 확인
- 사이트가 열리면 성공!
- 안 열리면 위의 문제 해결 단계를 따라하세요

## 💡 팁

### QR 코드로 쉽게 접속하기

1. 서버 IP 주소로 QR 코드 생성
2. 모바일로 스캔
3. 바로 접속!

### 북마크 저장
모바일 브라우저에 북마크를 저장하면 다음에 쉽게 접속할 수 있습니다.
