# 서버 실행 요구사항

## ⚠️ 현재 상황

### 컴퓨터를 켜두어야 하는 이유

현재 Next.js 서버가 **Mac mini에서 실행**되고 있습니다.

- ✅ **컴퓨터 켜져있음**: 사이트 접속 가능, 데이터 저장 가능
- ❌ **컴퓨터 꺼져있음**: 사이트 접속 불가, 데이터 저장 불가

## 🔄 작동 원리

```
[사용자] → [Mac mini 서버] → [NAS 데이터베이스]
           (켜져있어야 함)    (항상 저장됨)
```

- 서버가 실행 중이어야 요청을 처리할 수 있습니다
- 서버가 꺼지면 사이트에 접속할 수 없습니다
- 데이터는 NAS에 저장되지만, 서버가 없으면 저장할 수 없습니다

## ✅ 해결 방법

### 방법 1: 컴퓨터를 항상 켜두기 (현재 방법)

**장점**:
- 간단함
- 추가 설정 불필요

**단점**:
- 전기 사용
- 컴퓨터가 항상 켜져있어야 함

**설정**:
- 시스템 설정 > 에너지 절약 > 컴퓨터 절전 방지
- 또는 화면만 꺼지도록 설정

### 방법 2: NAS에서 직접 서버 실행 (권장)

**Synology NAS에서 Node.js 실행**

**장점**:
- NAS는 항상 켜져있음
- 컴퓨터를 꺼도 사이트 작동
- 전력 소비 적음

**단점**:
- Synology NAS 설정 필요
- Docker 또는 Node.js 설치 필요

**설정 방법**:
1. Synology NAS에 Docker 설치
2. Node.js 컨테이너 실행
3. 프로젝트 파일을 NAS에 복사
4. 서버 실행

### 방법 3: 다른 서버 사용

- 라즈베리 파이 같은 저전력 서버
- 클라우드 서버 (AWS, Azure 등)
- 항상 켜져있는 다른 컴퓨터

## 🚀 NAS에서 실행하기 (추천)

### Synology Docker 사용

1. **Docker 설치** (Synology Package Center)
2. **Node.js 이미지로 컨테이너 생성**
3. **프로젝트 파일 마운트**
4. **서버 실행**

### 간단한 방법: SSH로 NAS에서 실행

1. Synology NAS에 SSH 활성화
2. Node.js 설치
3. 프로젝트 복사
4. 서버 실행

## 📊 비교

| 방법 | 컴퓨터 필요 | 전력 소비 | 설정 난이도 |
|------|------------|----------|------------|
| 현재 (Mac mini) | 항상 켜야 함 | 높음 | 쉬움 |
| NAS에서 실행 | 불필요 | 낮음 | 중간 |
| 클라우드 서버 | 불필요 | 없음 | 중간 |

## 💡 추천

**현재 상황**: 컴퓨터를 항상 켜두어야 합니다.

**개선 방법**: NAS에서 직접 서버를 실행하면 컴퓨터를 꺼도 됩니다.
